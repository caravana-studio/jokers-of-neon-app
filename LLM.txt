JOKERS OF NEON - LLM REFERENCE
Generated from repo source data.

GAME OBJECTIVE
- Reach each round target score by playing poker hands and progress through level maps.
- Build stronger decks by buying cards, modifiers, specials, power-ups, and loot boxes in stores.
- Survive escalating difficulty, including Rage rounds with temporary debuffs.
- Source summary: README + tutorials + in-game localization.

CORE LOOP
- Round: play/discard from hand to reach target score.
- If target is reached: go to map/store and invest coins in upgrades.
- If target is not reached and no hands remain: run ends.
- README notes a classic setup of 5 hands + 5 discards per round at run start.

STORE SECTIONS (DYNAMIC LAYOUT IDs)
- traditionals: traditional + neon card offers
- modifiers: modifier card offers
- specials: special card offers
- power-ups: consumable boosts (points/multi)
- loot-boxes: randomized packs
- level-up-table: poker hand level upgrades (points/multi growth)
- burn: remove a card from your deck
- unlock-slot: buy extra special-card slot (if below max)
- reroll: refresh current store inventory for a coin cost

ITEMS (NON-CARD SHOP MECHANICS)
- Burn item: permanently remove one card from deck.
- Special slot item: unlock one additional special card slot.
- Poker hand level-up item: increase selected hand level (raises points/multi).
- Reroll: re-randomize store inventory.
- Power-up slots: game config default max 4 if config fetch unavailable.
- Special-card slots: game config default max 7 if config fetch unavailable.

RAGE ROUNDS - HOW THEY WORK
- Rage rounds are separate map nodes (NodeType.RAGE) that apply one or more rage effects for that round.
- Active rage card ids are exposed in game round state as round.rages.
- Rage node payload encodes two numbers: round_index = floor(data / 2^32), power = data mod 2^32.
- Rage nodes are labeled as Intermediate, Final, or Final Boss in UI.
- Boss level constant in frontend: BOSS_LEVEL=3.
- Rage effects are typically temporary, round-scoped debuffs (see Rage card list above).

MAP MODEL
- Map is fetched from on-chain map_system.getLevelMap(gameId).
- Node types: ROUND, STORE, RAGE, REWARD, NONE (challenge support exists in UI enum).
- Node data semantics:
  - STORE node data => shop_id
  - ROUND node data => round number
  - RAGE node data => encoded round/power (see Rage section)
- Each node has child connections; UI renders a DAG with branch choices.
- Legend labels in app include: normal round, rage (intermediate/final), and six shop types.

SCORING BASICS
- Final play score uses Points x Multi model.
- Traditional card base values from card texts: 2-10 pip values, J/Q/K=10, A=11.
- Joker card text: can substitute any traditional card; grants +100 points and +1 multi when played.
- Neon cards generally provide doubled point profile plus +1 multi over traditional equivalents.
- Modifiers, specials, power-ups, and rage effects can alter scoring and legal hand composition.

TEMPORARY VS PERMANENT SPECIALS
- Store info defines temporary specials as auto-removed after 3 rounds.
- Permanent specials persist until sold/removed.

KNOWN DATA NOTES
- Loot box translation provides human-readable odds/details for ids 1-13.
- PACK_RATES currently provides explicit slot-level probability matrices for pack ids 1,2,3,4,5,6,8,11,12,13,21,22,23,24,25,26.
- Rarity mapping files may not include every card id present in localization catalogs (new/legacy content coexist).
